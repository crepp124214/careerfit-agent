# CareerFit 用户体验综合评价报告

**评估日期**：2026-05-04
**评估角色**：普通计算机应届生用户
**测试环境**：Chrome 桌面端（1440×900）+ iPhone 13 移动端 + iPad Pro 平板端
**测试范围**：全部 8 个页面、核心闭环全流程、响应式三端

---

## 一、实用性评价 ⭐⭐⭐☆☆（3/5）

### ✅ 做得好的地方

1. **核心闭环走通**：创建岗位 → 创建简历 → 执行分析 → 查看报告 → 面试训练 → 学习任务，主路径确实可以端到端完成，这是最基本也最重要的。
2. **Next Best Action 有价值**：报告和工作台都会提示"下一步建议"（如"创建下一版简历并重新分析"），对不知道下一步该干什么的应届生很友好，不是装饰性功能。
3. **面试训练闭环完整**：按技能维度生成面试题，支持三维筛选（技能/类别/难度），有进度追踪和练习笔记，可以从报告页直接跳转"开始面试训练"。
4. **版本对比有意义**：能看到两版简历之间的文本差异（新增/删除行数）和分数变化，帮助理解优化效果。
5. **报告导出可用**：Markdown 导出和打印/PDF 功能都有，方便保存和分享。
6. **隐私提示到位**：底部固定"JD 与简历文本已脱敏"提示，让用户安心。

### ❌ 严重问题

#### 🔴 P0：评分维度严重缺失

**现象**：我输入了一个包含 SQL、Python、数据可视化、统计方法、机器学习、A/B 测试等多维要求的"数据分析师"JD，但报告**只产出了 1 个维度（Python 30分）**，总分 27 分。

**影响**：这完全不能反映简历和岗位的真实匹配度。这位候选人在 SQL、数据分析、A/B 测试等方面都有强匹配，却被全部忽略。**这是最致命的实用性问题，直接让"匹配分析"这个核心功能失去了可信度。** 用户看到 27 分会认为自己和岗位完全不匹配，但实际上匹配度可能超过 60%。

**期望**：应该根据 JD 要求产出 5-8 个维度的评分，每个维度都有独立的 JD 证据、简历证据和知识库标准。

#### 🔴 P0：RAG 知识库检索不相关

**现象**：数据分析师岗位检索到的知识库内容是"大模型应用学习资源""FastAPI 技能定义""后端开发岗位画像"，这些和数据分析岗完全无关。

**影响**：知识库标准部分展示了后端开发的内容，误导用户以为这些是数据分析岗位的核心要求。RAG 检索应该返回统计学、数据分析方法论、SQL 优化、数据可视化等知识。

**根因推测**：可能是 embedding 模型对中文岗位描述的语义理解不足，或者种子知识库中缺少数据分析相关文档，或者检索时的相似度阈值设置不当。

#### 🟡 P1：简历建议过于笼统

**现象**：只有一条"强化 Python 表达"的建议，没有针对 SQL、数据可视化、统计方法等维度的具体优化指导。

**影响**：对于应届生来说，需要更细粒度的建议，比如"在 SQL 项目中补充窗口函数的具体使用场景"或"将'使用过 Echarts'改为'使用 Echarts 构建了包含 5 种图表的实时数据看板'"。

#### 🟡 P1：面试题目模板化

**现象**：所有面试题都是"请说明你在 X 上最具体的一次实践"的统一模板，缺乏针对性。

**影响**：好的面试题应该根据技能类别不同而变化——SQL 应该问"如何优化慢查询"，统计方法应该问"如何设计 A/B 测试并评估统计显著性"，数据可视化应该问"如何选择合适的图表类型"。

#### 🟡 P1：学习任务内容薄弱

**现象**：只有两条学习任务，且都是"完成一个使用 X 的小型可运行项目"的模板，没有具体的学习路径、资源推荐或时间规划。

**影响**：应届生需要的是"先学什么→再学什么→用什么资源→做什么练习→怎么验证"的结构化学习路径，而不是泛泛的"做个项目"。

## 二、易用性评价 ⭐⭐⭐⭐☆（4/5）

### ✅ 做得好的地方

1. **导航清晰**：左侧导航栏图标+文字，8 个入口一目了然，当前页高亮明显，图标选用恰当（工作台=仪表盘、岗位=公文包、简历=文件等）。
2. **操作流程直觉化**：工作台上选岗位→选简历→点分析，三步完成，不需要额外学习成本。
3. **表单设计合理**：创建岗位和简历的模态框，必填项标红星，占位符有具体示例（如"例如：后端开发工程师"），保存按钮在未填必填项时禁用，防止误操作。
4. **状态反馈完整**：加载中有"加载中…"提示，错误有重试按钮，空状态有引导文案（如"完成一次匹配分析后，可以从报告页创建面试训练"）。
5. **筛选器好用**：面试训练的技能/类别/难度三维筛选器，操作流畅，结果即时更新。
6. **主题切换即时**：暗色/亮色/跟随系统三种主题，切换后全站即时生效，无需刷新。
7. **连接状态可见**：底部固定"已连接"状态指示，让用户知道后端是否可用。
8. **移动端汉堡菜单**：小屏幕下侧边栏自动收起，点击汉堡按钮展开导航，交互自然。

### ❌ 需要改进的地方

#### 🟡 P1：时间戳不友好

**现象**：岗位和简历列表显示的是 ISO 格式时间戳（如 `2026-05-03T18:43:02.535941`），普通用户根本看不懂。

**建议**：显示为"2 小时前"或"2026-05-03 18:43"这种人类可读格式。

#### 🟡 P1：版本名称显示混乱

**现象**：简历列表中"v1 — 面向数据分析岗 — v1"出现了两次 v1，让人困惑。看起来是用户输入的版本名称和系统自动生成的版本标签拼接的问题。

**建议**：如果用户已指定版本名称，就不要再追加系统版本标签；或者用更清晰的方式区分，如"v1 — 面向数据分析岗（版本 1）"。

#### 🟡 P1：报告页缺少返回入口

**现象**：从工作台跳到报告后，没有返回工作台的明显入口（只能点侧边栏"工作台"），容易迷失上下文。

**建议**：在报告页顶部加面包屑导航，如"工作台 > 分析报告"。

#### 🟡 P1：Agent Trace 信息过于技术化

**现象**：`jd_parser`、`rag_retriever`、`match_scorer` 这些节点名称对普通用户毫无意义。

**建议**：用通俗描述替代，如"解析岗位要求""检索知识库""计算匹配分数"。保留技术名称作为 tooltip 或展开详情。

#### 🟢 P2：分析确认界面交互不明确

**现象**：点击工作台"开始分析"后出现的确认界面，"开始分析"按钮位置不明显，我第一次点到了 Next Best Action 的"开始分析"按钮而非确认对话框的。

**建议**：确认对话框应该有遮罩层，或者用更醒目的样式区分。

#### 🟢 P2：历史趋势图表缺失

**现象**：页面显示"趋势图表功能即将上线"，但这是核心功能，没有图表的趋势页价值大打折扣。当前只有数字列表，用户很难直观感受分数变化趋势。

## 三、美观性评价 ⭐⭐⭐⭐☆（4/5）

### ✅ 做得好的地方

1. **设计系统一致**：全站使用统一的设计 token（颜色、圆角、间距），视觉风格一致，没有"拼凑感"。组件命名遵循 BEM 规范，代码和视觉都井然有序。
2. **Next Best Action 卡片醒目**：左侧有彩色竖条装饰，标题层级清晰（eyebrow → headline → CTA），是整个页面的视觉焦点，符合"引导用户行动"的设计意图。
3. **评分环形图精致**：SVG 圆环进度条 + 中心数字 + 颜色编码（红色=高风险），信息密度和美观度都不错。
4. **风险标签双通道**：高风险用红色圆点+文字"高风险"，低风险用绿色圆点+文字"通过"，满足色盲友好要求，符合 WCAG AA。
5. **暗色主题质量高**：暗色模式下对比度充足，没有"灰蒙蒙"的感觉，侧边栏和内容区层次分明，文字可读性好。
6. **动效克制但有**：页面入场有 `animate-in` 动画，交错延迟（stagger）让视觉节奏舒适，不会过度花哨。
7. **移动端适配到位**：iPhone 13 和 iPad Pro 上布局都正常，侧边栏自动收起为汉堡菜单，内容区自适应宽度。
8. **品牌标识简洁**：CF logo + CareerFit 文字，侧边栏顶部，辨识度高且不喧宾夺主。

### ❌ 需要改进的地方

#### 🟡 P1：岗位/简历列表过于朴素

**现象**：纯文字列表，没有卡片阴影、图标或色彩区分，和上方 Next Best Action 卡片形成视觉落差。

**建议**：加卡片效果（圆角+阴影+hover 态），公司名用小字灰色显示，技能标签用彩色 pill 展示。

#### 🟡 P1：报告页信息密度过高

**现象**：评分明细、知识库标准、简历建议、面试问题、Agent Trace 全部纵向堆叠，缺乏视觉分组和呼吸感。知识库标准部分展示了 3 段长文本，阅读压力很大。

**建议**：用 Tab 分隔（评分 | 建议 | 面试 | Trace），或用折叠面板让用户按需展开。知识库标准应该只显示摘要，点击展开全文。

#### 🟢 P2：模态框偏小

**现象**：创建岗位/简历的模态框宽度 480px，在大屏上显得局促，特别是 JD 和简历的 textarea 太窄，输入长文本时不舒服。

**建议**：大屏上宽度可以到 640px 或 720px，textarea 行数增加到 10-12 行。

#### 🟢 P2：空状态缺少插图

**现象**：空列表只有文字提示，视觉上显得空洞。

**建议**：加个简单的 SVG 插图（如空文件夹、空文档），会更友好、更有温度。

#### 🟢 P2：进度条颜色单一

**现象**：面试训练的进度条只有一种颜色（主色调），不能直观反映完成情况。

**建议**：根据完成百分比变色——<30% 红色，30-70% 黄色，>70% 绿色。

## 五、优先修复建议

| 优先级 | 问题 | 影响范围 | 修复方向 |
|--------|------|----------|----------|
| 🔴 P0 | 评分维度缺失 | 核心功能 | 修复 match_scorer 逻辑，确保根据 JD 要求产出多维度评分 |
| 🔴 P0 | RAG 检索不相关 | 核心功能 | 补充数据分析类种子文档，调优 embedding/检索参数 |
| 🟡 P1 | 时间戳格式化 | 全站 | 前端统一用 dayjs 格式化为相对时间或本地格式 |
| 🟡 P1 | 面试题去模板化 | 面试训练 | LLM prompt 增加技能类别→题型映射 |
| 🟡 P1 | 简历建议细粒度 | 报告 | 每个缺口维度产出独立建议，含具体修改示例 |
| 🟡 P1 | 报告页信息分组 | 报告 | 用 Tab 或折叠面板重构信息架构 |
| 🟡 P1 | Agent Trace 通俗化 | 报告 | 节点名映射为中文描述 |

| 🟢 P2 | 趋势图表 | 历史 | 引入轻量图表库（如 Chart.js） |
| 🟢 P2 | 模态框尺寸 | 表单 | 大屏上加宽到 640px+ |

## 六、总结

CareerFit 的**框架和体验基础是扎实的**——设计系统一致、操作流程直觉、响应式到位、暗色主题质量高。作为一个求职成长工作台，它的页面覆盖完整，核心闭环可以走通。

但**核心分析质量是当前最大的短板**。评分维度缺失和 RAG 检索不相关这两个 P0 问题，直接动摇了"匹配分析"这个产品核心价值的可信度。一个求职工具如果给出的分析结果不可信，用户就不会持续使用。

建议**优先解决 P0 问题**，让匹配分析真正能产出多维度、有证据支撑、知识库相关的可信报告，然后再打磨 P1/P2 的体验细节。
